# Customer Support Chatbot Case Study

## Objective

Design a customer support chatbot prototype that helps small businesses answer repeated customer questions faster and more consistently.

## Business Problem

Small businesses often receive repeated questions about product information, order status, shipping fee, return policy, store opening hours, and basic troubleshooting. Answering the same questions manually can take time and reduce support quality during busy hours.

## Proposed Chatbot Solution

The chatbot uses a structured FAQ dataset and prompt templates to answer common questions. When a question is unclear, it asks a short clarification question instead of inventing unsupported information.

## Supported Intents

| Intent | Description |
|---|---|
| `product_information` | Answer basic product questions |
| `shipping_fee` | Explain delivery/shipping fee rules |
| `return_policy` | Explain return conditions |
| `store_hours` | Provide opening hours |
| `order_support` | Guide the customer to provide order information |
| `unknown` | Ask clarification |

## Deliverables

- FAQ dataset format
- Chatbot conversation flow
- Prompt template
- Example responses
- Testing checklist
- User guide for business users

## Example Conversation

**User:** How much is the shipping fee?  
**Bot:** Shipping fee depends on your delivery location and order value. Could you provide your district or address so I can help estimate it?

## Evidence Value

This case study shows practical chatbot planning, business use case understanding, prompt design, and documentation ability.
